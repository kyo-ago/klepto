    Kleptoとは？

    JavaScript, CSSの開発者向けのLocal Proxyです。

    何ができるの？

    現在以下の機能が実装されています。

    • 通信内容の閲覧（network list）
    • 通信内容の差し替え（auto responder）
    • Chrome developer tools上で書き換えた内容のローカルファイルへの反映（auto save）
    • ローカルファイルが更新された場合にブラウザをリロード（auto reload）
    （注　現在Windows環境では「通信内容の閲覧」と「通信内容の差し替え」が正しく行えません）

    動作環境

    MacOSX版Chrome version 25以上（2013/03/01の安定版最新）

    現在Windows環境では「network list」と「auto responder」が動作しません（1ページ通信するとそれ以上通信できなくなる）
    「auto save」と「auto reload」は動作するので、「network list」と「auto responder」を行いたい場合Fiddler Web Debuggerをおすすめします。

    使い方

    • Klepto proxy（Chrome ウェブストア）をインストール
    • Klepto-extension（Chrome ウェブストア）をインストール
    • Chromeの新規タブから「Klepto proxy」を起動
    • 起動したKlepto proxy windowにファイル、ディレクトリをD&Dで渡す
    • D&Dで渡したローカルファイルをエディタ等で編集
    • Chrome developer toolsのSourcesタブでD&Dで渡したファイルを編集

    制限事項

    • 動作はそこそこ遅いです
    • SSLページは差し替えられません
    • auto reloadは「今開いてるタブをリロードする」だけです。編集したファイルを読み込んでいるかどうかの確認は行なっていません
    • auto reloadは5秒毎に更新時間を確認しています。リロードは最大5秒遅延します
    • 現状Kleptoのwindowを開いている場合のみ動作します（修正予定）
    • ブラウザが終了するとauto responderの設定が消えます（仕様的にどうしようもないと思う）

    内部実装

    network list、auto responderはKlepto proxy(Chrome apps)上にsocket APIを使ってHTTP Proxyを構築し通信を中継しています。

    auto save、auto reloadはKlepto proxy(Chrome apps)上にWebSocketサーバを構築し、Klepto extension(Chrome extension)からWebSocketで接続してコマンドのやり取りを行なっています。
    （WebSocketサーバはOrignヘッダを確認しているので、Klepto extension以外からのWebSocket通信は受け取りません）
